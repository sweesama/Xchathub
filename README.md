# XChat Hub

Unofficial community resource site for XChat. The project is built to capture search demand around XChat launch status, Android availability, FAQs, comparisons, and future group discovery.

## What This Project Is

XChat Hub is a static Astro site for `xchat.directory` with three main goals:

1. Track XChat launch and Android availability.
2. Publish SEO/AEO-friendly pages like FAQ, launch countdown, and app comparisons.
3. Route users into the Telegram channel/community and, later, a group directory plus listing monetization.

## Stack

- Astro 4
- Native CSS
- Vercel deployment
- Partytown for GA script isolation
- `@resvg/resvg-js` for generating PNG assets from SVG sources

## Single Source Of Truth

The launch-sensitive product facts now live in:

- [src/data/xchat.json](f:\windsurf\xchat\src\data\xchat.json)

This file is the canonical source for:

- App Store URL
- Telegram channel URL
- Launch ISO timestamp
- Launch date display text
- Short launch date display
- iOS requirement
- Android status
- Group size limit

Do not manually edit those values across pages unless there is a specific reason not to use the shared data module.

## Maintenance Workflow

When launch details change:

1. Update [src/data/xchat.json](f:\windsurf\xchat\src\data\xchat.json).
2. Run `npm run build`.
3. Sanity-check homepage, FAQ, launch countdown, and `public/llms.txt`.

Detailed checklist:

- [docs/launch-checklist.md](f:\windsurf\xchat\docs\launch-checklist.md)
- [docs/app-store-monitoring.md](f:\windsurf\xchat\docs\app-store-monitoring.md)
- [docs/group-research-workflow.md](f:\windsurf\xchat\docs\group-research-workflow.md)
- [docs/group-publishing-workflow.md](f:\windsurf\xchat\docs\group-publishing-workflow.md)

When shared copy or metadata changes:

1. Check [src/layouts/Layout.astro](f:\windsurf\xchat\src\layouts\Layout.astro).
2. Check [src/components/Header.astro](f:\windsurf\xchat\src\components\Header.astro).
3. Rebuild and verify.
